# DailyKit Product Brain v9.6

## Phase Update

DailyKit v9.6 adds a visible app-update flow on top of the new settings, diagnostics, and offline maintenance system.

## New In This Version

- Visible service-worker update toast instead of silent reload behavior
- In-app changelog modal so users can review what changed before updating
- Completed product-brain snapshot coverage for the settings and release-management phase
- Version bump across app shell, manifest, service worker cache, tool manifest, and translations

## Why This Matters

This version makes releases easier to trust. Users can now see when a fresh version is ready, understand what changed, and update the installed/offline app intentionally instead of being surprised by a silent refresh.

## Current Phase Summary

The app now includes:

- Offline-first guest workspace flow
- Search and command center
- Expenses, Borrowed, Grocery, Habits, Notes, and Subscriptions
- CSV backup and restore
- Reports, onboarding, reminders, diagnostics, and settings
- Storage versioning, event bus, lifecycle-aware tools, and category learning

## Next Recommended Step

For v9.7, the strongest next upgrade is a lightweight settings toggle layer for things like onboarding visibility, backup reminders, and command-search behavior, so more of DailyKit becomes user-configurable without changing the architecture.
